## Why

现有简历优化功能是割裂的：JD匹配分析、结构检查、深度润色、STAR改写——四个独立tab，用户自己决定用哪个。但真实世界中，简历经过一条筛选漏斗（ATS → HR初筛 → 业务主管 → 决策层），每一层看的东西完全不同。用户不知道自己的简历会在哪一层被筛掉、为什么。需要一个**综合体检**功能，模拟这条漏斗，给出"你的简历会在第几层被淘汰"的诊断。

## What Changes

- 新增"简历体检"功能（新tab或替换现有结构检查tab），一次调用返回多维度评估报告
- 模拟4层筛选漏斗分别打分：ATS过筛率、HR初筛通过率、业务主管评分、风险信号检测
- 新增以下评估维度（现有结构检查未覆盖）：
  - **量化密度**：扫描每条经历，标记缺乏数字的条目，统计量化覆盖率
  - **动词强度**：检测弱动词（参与/协助/负责），建议替换为强动词
  - **成果导向比**：区分"做了什么"vs"做成了什么"
  - **红旗信号**：跳槽频率、空窗期、职级描述不匹配、技能堆砌
  - **职业叙事**：成长曲线是否递增、换公司逻辑是否通顺
  - **ATS关键词覆盖**：通用技术关键词是否在标准位置出现
- 输出结构化的体检报告：综合评分 + 分漏斗评分 + 红旗/警告/亮点分类 + 逐条批注
- 逐条批注保持和现有结构检查一致的交互（引用原文 → 问题 → 建议 → 一键应用改写）

## Capabilities

### New Capabilities
- `resume-health-checkup`: 综合简历体检——模拟ATS/HR/主管/风险四层漏斗，一次调用返回多维度评估报告，包含量化密度、动词强度、成果导向、红旗信号、职业叙事等维度，输出结构化体检报告

### Modified Capabilities
- `resume-structure-analysis`: 现有结构检查tab将整合为体检报告中的一个维度（信息架构），不再作为独立tab存在。后端接口保留兼容，前端tab合并

## Impact

- **后端**：新增 `POST /api/resume/health-checkup` 端点，新增 `HealthCheckupRequest/Response` DTO，新增 prompt 模板 `resume/health-checkup.md`，`ResumeAiService` 新增分析方法
- **前端**：`ResumeOptimizerView.vue` 新增或改造 tab，新增体检报告 UI 组件，`resumeStore` 新增体检相关 state/actions
- **Prompt**：新增 `health-checkup.md`，设计为单次调用返回完整 JSON 结构
- **现有功能**：结构检查接口保留不变，前端入口合并到体检tab中作为子模块
