## Context

当前面试演练室每次只生成一道题，通过 SSE 流式获取点评和追问。批量出题模式是全新的交互方式：一次性生成多道题+答案，用户快速浏览、点击展开，适合刷题场景而非模拟面试场景。

现有架构：
- `InterviewController` 提供 `POST /api/interview/question` 生成单题，返回结构化 JSON
- `InterviewAiService` 渲染 prompt 模板调用 AI
- 前端 `interviewStore` 管理单题状态机

约束：
- AI 模型单次输出有长度限制，大批量题目需分批请求
- 智谱 GLM 系列有速率限制（429），需控制请求频率

## Goals / Non-Goals

**Goals:**
- 一次生成 10/20/50/100 道面试题，每题附带参考答案
- 题目列表默认只显示题目，点击展开/折叠答案
- 支持保存单题或批量保存到 Obsidian 知识库
- 前端独立页面，侧边栏新增导航入口

**Non-Goals:**
- 不实现批量答题评分（用户自测，不做关键词分析）
- 不实现流式输出（批量出题返回完整 JSON，不做 SSE）
- 不实现追问挑战模式
- 不改变现有面试演练室功能

## Decisions

### 1. 后端分批请求，前端展示合并结果

**选择**: 后端按每批 10 道题分批调用 AI，并行请求（最多 3 个并发），合并结果后一次性返回前端。

**备选方案**:
- 前端分批请求，逐批渲染 → 前端逻辑复杂，需要管理多批次状态
- 单次请求生成所有题目 → 100 题时 AI 输出超长，容易截断或格式错乱

**理由**: 分批控制单次 AI 输出长度在可控范围内（每批 10 题约 3000-5000 字），并发提高速度，后端合并简化前端逻辑。

### 2. 新增批量出题 prompt 模板

**选择**: 新建 `backend/prompts/interview/batch-question.md`，要求 AI 一次输出 10 道题+答案的结构化 JSON。

**理由**: 与现有单题 prompt 分离，避免互相影响。批量 prompt 要求输出 JSON 数组格式，每项包含 `{question, answer, keywords}`。

### 3. 前端使用独立的 Store 和页面

**选择**: 创建 `rapidQuestionStore.ts` 和 `RapidQuestionView.vue`，不复用 interviewStore。

**理由**: 面试演练室是单题状态机，批量刷题是列表+折叠模式，状态管理逻辑完全不同。独立 store 避免互相污染。

### 4. 题目列表的展开/折叠交互

**选择**: 使用 CSS transition 实现平滑展开折叠，默认全部折叠。每题有独立的展开状态。

**理由**: 简单直观，用户可以快速扫视所有题目，选择性查看答案。不需要复杂的虚拟滚动（100 题的数据量 DOM 可以承受）。

## Risks / Trade-offs

- **[AI 输出格式不稳定]** → 使用结构化 JSON 输出 + 后端解析校验，格式错误时降级为逐题生成
- **[大量题目生成耗时]** → 50/100 题需要 5-10 批 AI 调用，前端显示进度条，支持后台生成
- **[速率限制 429]** → 并发控制在 3 以内，请求间加 500ms 间隔，429 时自动重试（最多 2 次）
- **[Token 成本]** → 用户主动选择数量，默认 10 题，大数量时有提示
